<BionicText method="advanced" intensity="medium" class="highlight">
A pantograph is an elegant mechanical linkage that scales motion through similar triangle geometry. It can enlarge, reduce, or exactly copy a path traced by one point as output at another point, with applications ranging from engraving to drafting tools.
</BionicText>

### Real-World Applications

The pantograph appears in diverse engineering applications:

:::note[Common Applications]
- **Engraving and routing machines** - Copying patterns at different scales
- **Drafting tools** - Enlarging or reducing technical drawings
- **Manufacturing equipment** - Motion scaling in production lines
- **Educational demonstrators** - Teaching similar triangles and geometry
- **Robotic manipulators** - Coordinated motion control
- **Historical devices** - Pre-computer enlargement/reduction tools
:::

### The Engineering Problem

> **Design Challenge:** Given a path traced by an input stylus or pointer, reproduce that path at a different scale while maintaining geometric similarity and smooth motion.

## üìö Mechanism Fundamentals

### Pantograph Principle

<Tabs>
  <TabItem label="Similar Triangles">

    **Core Geometric Principle:**

    A pantograph consists of four links forming a parallelogram-like structure with specific length ratios.

    **Key Setup:**
    - One pivot is fixed (O)
    - Input point (I) traces a path
    - Output point (P) is positioned on an extension of link OI

    **Magic Relationship:**

    $$\frac{OP}{OI} = k \text{ (constant ratio)}$$

    This ratio k is the **scaling factor**!

  </TabItem>
  <TabItem label="Scaling Factor">

    <Card title="‚öôÔ∏è Understanding Scaling Ratio k" icon="setting">

    **k > 1: Enlargement**
    - Output motion larger than input
    - Example: k = 2 means output is 2√ó input
    - Used for: Enlarging drawings, patterns

    **k = 1: Exact Copy**
    - 1:1 pantograph
    - Output exactly matches input
    - Used for: Duplicating patterns

    **k < 1: Reduction**
    - Output smaller than input
    - Example: k = 0.5 means output is half input
    - Used for: Miniaturization, engraving

    </Card>

  </TabItem>
  <TabItem label="Geometric Requirements">

    **Link Arrangement:**

    ```
    - OA and OB: Two links from fixed pivot O
    - A and B: Moving pivots
    - AB: Connecting link
    - I (input): On link OA extended
    - P (output): On link OB extended
    ```

    **Critical Length Relationships:**

    $$\frac{OA}{OI} = \frac{OB}{OP} = \frac{AB}{IP} = k$$

    :::tip[Why This Works]
    The parallelogram OAIB ensures that triangles OIA and OPB remain similar as the mechanism moves. This geometric similarity is what maintains the constant scaling ratio throughout the entire range of motion!
    :::

  </TabItem>
</Tabs>

## üí° Part 2: Parametric Design Strategy

<BionicText method="advanced" intensity="medium" class="highlight">
The pantograph is our first ratio-driven mechanism where geometric relationships are more complex. We'll use a master sketch to define the kinematic layout, then reference it from all parts. This ensures geometric consistency and simplifies the parametric control to just two values: base length and scaling ratio.
</BionicText>

### Our Design Approach

<Card title="üéØ Ratio-Based Control Philosophy" icon="rocket">

We'll control the **entire pantograph mechanism** with just two parameters:

**BaseLength** = 100 mm
**ScalingRatio** = 2

All other dimensions calculated automatically using expressions!

$$\text{InputLength} = \text{BaseLength} \times \text{ScalingRatio}$$

$$\text{OutputLength} = \text{BaseLength} \times \text{ScalingRatio}^2$$

$$\text{CrossLink} = \text{BaseLength} \times \text{ScalingRatio}$$

This is parametric design at its most powerful.

</Card>

### Design Workflow

<Tabs>
  <TabItem label="Components">

    **Our Pantograph Configuration:**

    - Fixed pivot O at origin
    - Scaling ratio $$k = 2:1$$ (output is 2√ó input)
    - Input stylus at point I
    - Output tracer at point P

    **Link Naming:**

    1. **Base** - Fixed ground (mounting plate)
    2. **Link OA** - From fixed pivot O to moving pivot A
    3. **Link OB** - From fixed pivot O to moving pivot B
    4. **Link AB** - Connecting moving pivots A and B
    5. **Input Arm (OI)** - From O through A to input point I
    6. **Output Arm (OP)** - From O through B to output point P

  </TabItem>
  <TabItem label="Build Sequence">

    <Steps>

    1. Create Spreadsheet (parameters and formulas)

    2. Create Master Sketch (kinematic layout)

    3. Create Link OA

    4. Create Link OB

    5. Create Link AB

    6. Create Input Arm (OI)

    7. Create Output Arm (OP)

    8. Create Base

    9. Assemble with constraints

    10. Test parametric control

    </Steps>

  </TabItem>
  <TabItem label="Why Master Sketch?">

    :::note[Master Sketch Benefits]
    **For Complex Mechanisms like Pantographs:**
    - Multiple parts share exact geometric points
    - Similar triangle relationships must be maintained
    - Master sketch ensures all parts stay coordinated
    - Change one sketch, all parts update correctly

    **Think of it as:** The master sketch is the kinematic blueprint; individual parts are the physical implementation.
    :::

  </TabItem>
</Tabs>

## üé® Part 4: Creating the Master Sketch

### Design Intent

<Card title="üó∫Ô∏è Master Sketch Purpose" icon="document">

The master sketch defines the **kinematic layout** of the pantograph:

- All critical points (O, A, B, I, P)
- All link lengths with parametric control
- Geometric relationships between components

Think of it as the "blueprint" that all parts will reference.

</Card>

### Step-by-Step: Master Sketch

<Steps>

1. **Create standalone sketch**

   - Ensure you're in **Part Design workbench**
   - Click **Create Sketch** (don't create a Body first!)
   - Select **XY_Plane**
   - This sketch won't belong to any body - it's purely reference geometry

</Steps>

### Drawing the Kinematic Layout

<Tabs>
  <TabItem label="Fixed Pivot O">

    **The center of rotation:**

    The origin (0, 0) is our fixed pivot point O.

    No need to draw anything - the origin marker is already there!

  </TabItem>
  <TabItem label="Link OA">

    **First base link:**

    <Steps>

    1. **Line tool** (press L)
    2. Click at **origin** (0, 0)
    3. Draw at approximately 45¬∞ upward-right
    4. Click to place endpoint
    5. Press **Escape**

    6. **Add length constraint:**
       - Distance/Length constraint tool
       - Click the line
       - Click **∆íx** button
       - Type: `Spreadsheet.BaseLength`
       - Enter

    7. **Add angle constraint:**
       - Angle constraint tool
       - Click the line
       - Type: `45` (degrees from horizontal)
       - Enter

    </Steps>

    Label this endpoint mentally as "A"

  </TabItem>
  <TabItem label="Link OB">

    **Second base link:**

    <Steps>

    1. **Line tool**
    2. Click at **origin**
    3. Draw horizontally to the right
    4. Click to place endpoint
    5. Escape

    6. **Make horizontal:**
       - Select the line
       - Press **H** key (or apply Horizontal constraint)

    7. **Add length constraint:**
       - Distance constraint
       - Click the line
       - ∆íx ‚Üí `Spreadsheet.BaseLength`
       - Enter

    </Steps>

    Label this endpoint as "B"

  </TabItem>
  <TabItem label="Cross Link AB">

    **Connecting link:**

    <Steps>

    1. **Line tool**
    2. Connect point A to point B
    3. Escape

    4. **Add length constraint:**
       - Distance constraint
       - Click the line
       - ∆íx ‚Üí `Spreadsheet.CrossLink`
       - Enter

    </Steps>

    This forms triangle OAB - the core parallelogram structure!

  </TabItem>
  <TabItem label="Input Arm OI">

    **Extension to input point:**

    <Steps>

    1. **Line tool**
    2. Start at **A** (endpoint of OA)
    3. Draw along the same 45¬∞ angle (extending OA)
    4. Click to place endpoint
    5. Escape

    6. **Collinear constraint:**
       - Select the new line
       - Select line OA
       - Apply **Collinear** constraint
       - This ensures they're on the same line!

    7. **Total distance O to I:**
       - Distance constraint
       - Click **origin**
       - Click **far endpoint** (point I)
       - ∆íx ‚Üí `Spreadsheet.InputLength`
       - Enter

    </Steps>

  </TabItem>
  <TabItem label="Output Arm OP">

    **Extension to output point:**

    <Steps>

    1. **Line tool**
    2. Start at **B**
    3. Draw horizontally right (extending OB)
    4. Click to place endpoint
    5. Escape

    6. **Collinear constraint:**
       - Select new line and line OB
       - Apply **Collinear** constraint

    7. **Total distance O to P:**
       - Distance constraint
       - Origin ‚Üí far endpoint (P)
       - ∆íx ‚Üí `Spreadsheet.OutputLength`
       - Enter

    </Steps>

  </TabItem>
  <TabItem label="Construction Mode">

    **Make everything reference geometry:**

    <Steps>

    1. Press **Ctrl+A** to select all geometry

    2. Press **G** to toggle construction mode

    3. All lines turn blue/dashed - they're now construction geometry

    </Steps>

    :::tip[Why Construction Mode?]
    Construction geometry is **reference only** - it won't be extruded into 3D. It's perfect for master sketches that define layout but aren't physical parts themselves.
    :::

    **Optional:** Add small circles (radius 2mm) at points O, A, B, I, P for visualization

  </TabItem>
  <TabItem label="Verify Constraints">

    **Check the solver:**

    Should show: **"Fully constrained"**

    **What's defined:**
    - O is at origin (coincident)
    - OA length and angle (45¬∞)
    - OB length and horizontal
    - AB length (cross link)
    - OI total length (input arm)
    - OP total length (output arm)

    :::caution[If Not Fully Constrained]
    Missing constraints might include:
    - Origin coincident with O
    - Horizontal constraint on OB
    - Angle constraint on OA
    - Collinear constraints on extensions
    :::

  </TabItem>
</Tabs>

<Steps>

1. **Close the sketch**

   Click **Close** button in toolbar

2. **Rename for clarity**

   - Find "Sketch" in the tree
   - Right-click ‚Üí Rename ‚Üí `MasterSketch`

</Steps>

**Master sketch complete!** This defines all key points for the entire mechanism.

## üî© Part 5: Creating Link OA

<InArticleAd />


### Design Intent

<Card title="‚öôÔ∏è Link OA Requirements" icon="document">

Link from fixed pivot O to moving pivot A:

- **Holes at O and A** for pin joints
- **Rectangular body** connecting the holes
- **References master sketch** for exact positioning

</Card>

### Step-by-Step: Link OA

<Steps>

1. **Create Body**

   - Part Design workbench
   - Click **Create Body** button
   - Right-click ‚Üí Rename ‚Üí `Link_OA`

2. **Create Sketch**

   - Select Link_OA body
   - Create Sketch ‚Üí **XY_Plane**

3. **Reference master sketch points**

   - Click **External Geometry** tool (or press E)
   - In the tree, find and select **MasterSketch**
   - Click on points **O** and **A** in the 3D view
   - They appear as **purple reference points**

   :::tip[External Geometry Magic]
   These purple points are **linked** to the master sketch! If the master sketch updates, these references update automatically. This is how we maintain geometric consistency across all parts.
   :::

4. **Draw the link profile**

   - **Circle** at point O: Radius constraint ‚Üí ∆íx ‚Üí `Spreadsheet.PinRadius`
   - **Circle** at point A: Radius constraint ‚Üí ∆íx ‚Üí `Spreadsheet.PinRadius`
   - **Rectangle** connecting them, symmetric about the OA line
   - **Width** dimension: ∆íx ‚Üí `Spreadsheet.LinkWidth`

5. **Close sketch**

6. **Pad the link**

   - Select the sketch
   - Click **Pad** tool
   - Length: ∆íx ‚Üí `Spreadsheet.LinkThickness`
   - OK

</Steps>

**Link OA complete!**

## ‚úèÔ∏è Part 8: Creating Input Arm (OI)

   :::tip[Design Tip]
   You might want to make this arm slightly narrower than LinkWidth to avoid interference with other links. Try LinkWidth - 2mm.
   :::

</Steps>

**Input Arm complete!**

## üß© Part 11: Assembly

<BionicText method="advanced" intensity="medium" class="highlight">
Assembly is where the pantograph comes to life! With six separate parts all sharing a common pivot point at O, proper constraint strategy is critical. We'll use axial alignment to create revolute joints while maintaining the parallelogram geometry.
</BionicText>

### Assembly Strategy

<Card title="üéØ Assembly Constraints Plan" icon="rocket">

1. **Base:** Fixed (ground link)
2. **Four links pivot at O:** Link_OA, Link_OB, InputArm, OutputArm
3. **Joint at A:** Link_OA, Link_AB, InputArm meet
4. **Joint at B:** Link_OB, Link_AB, OutputArm meet
5. **All motion in XY plane**

</Card>

### Creating the Assembly

<Tabs>
  <TabItem label="Setup">

    <Steps>

    1. **Switch to Assembly workbench**

       Use workbench dropdown

    2. **Create new assembly**

       Assembly ‚Üí Create Assembly

    3. **Add all parts**

       Drag from tree or use "Add Part" button:
       - Base
       - Link_OA
       - Link_OB
       - Link_AB
       - InputArm
       - OutputArm

    </Steps>

  </TabItem>
  <TabItem label="Fixing Base">

    <Steps>

    1. Select **Base** in assembly tree

    2. Click **Fixed constraint** button

    3. Base is now locked in place (ground link)

    </Steps>

    :::note[Why Fix the Base?]
    Every mechanism needs a ground reference. The base provides this fixed reference point from which all other motion is measured. Point O on the base becomes our fixed pivot.
    :::

  </TabItem>
  <TabItem label="Central Pivot at O">

    **All four moving links pivot at O:**

    <Steps>

    1. **Link_OA to Base:**
       - Select Link_OA hole axis at O
       - Select Base hole axis at O
       - **Axial Align** constraint

    2. **Link_OB to Base:**
       - Link_OB hole at O ‚Üí Base hole at O
       - Axial Align

    3. **InputArm to Base:**
       - InputArm hole at O ‚Üí Base hole at O
       - Axial Align

    4. **OutputArm to Base:**
       - OutputArm hole at O ‚Üí Base hole at O
       - Axial Align

    </Steps>

    All four links can now rotate about O!

  </TabItem>
  <TabItem label="Joint at A">

    **Three links meet at point A:**

    <Steps>

    1. **Link_OA to Link_AB:**
       - Link_OA hole at A ‚Üí Link_AB hole at A
       - Axial Align

    2. **InputArm to Link_OA:**
       - InputArm hole at A ‚Üí Link_OA hole at A
       - Axial Align

    </Steps>

    Point A is now a complete pin joint!

  </TabItem>
  <TabItem label="Joint at B">

    **Three links meet at point B:**

    <Steps>

    1. **Link_OB to Link_AB:**
       - Link_OB hole at B ‚Üí Link_AB hole at B
       - Axial Align

    2. **OutputArm to Link_OB:**
       - OutputArm hole at B ‚Üí Link_OB hole at B
       - Axial Align

    </Steps>

    Point B is now a complete pin joint!

  </TabItem>
  <TabItem label="Testing Motion">

    **Test the pantograph:**

    <Steps>

    1. **Try dragging the input arm** (at point I)

    2. **Observe the output arm** (at point P)

    3. **Verify scaling:**
       - Input moves 10mm ‚Üí Output should move 20mm
       - Ratio should be 2:1

    </Steps>

    :::tip[Troubleshooting Motion]
    **If mechanism won't move:**
    - Check that only Base is Fixed
    - All pin joints should be Axial Align (not Fixed!)
    - Remove any redundant constraints
    - Links may need Z-offset to avoid interference

    **If parts interfere:**
    - InputArm and OutputArm may need to be offset in Z
    - Or make them narrower (LinkWidth - 2mm)
    :::

  </TabItem>
</Tabs>

## ‚úÖ Part 12: Verifying the Scaling Ratio

<BionicText method="advanced" intensity="medium" class="highlight">
The true test of a pantograph is its scaling accuracy. We'll verify both the geometric construction and the parametric control to ensure the mechanism works correctly at any scaling ratio.
</BionicText>

### Verification Methods

<Tabs>
  <TabItem label="Manual Test">

    **Measure actual motion:**

    <Steps>

    1. **Mark starting position**

       - Note coordinates of input point I
       - Note coordinates of output point P
       - Record or screenshot

    2. **Move input 10mm horizontally**

       - Drag input arm to move I exactly 10mm right
       - Use FreeCAD measurement tools if needed

    3. **Measure output displacement**

       - Check new position of P
       - Calculate displacement
       - Should be 20mm (2√ó input) with k=2

    4. **Test vertical motion**

       - Move I 10mm upward
       - P should move 20mm upward

    </Steps>

    ‚úÖ **Success indicator:** Output displacement = k √ó Input displacement

  </TabItem>
  <TabItem label="Parametric Test">

    **Change the scaling ratio:**

    <Steps>

    1. **Open Spreadsheet**

       Double-click Spreadsheet in tree

    2. **Change ScalingRatio**

       - Click cell B2
       - Change from `2` to `3`
       - Press Enter

    3. **Recompute**

       Press **Ctrl+R** or click Recompute button

    4. **Observe automatic updates:**

       - InputLength now = 300mm (100 √ó 3)
       - OutputLength now = 900mm (100 √ó 3¬≤)
       - CrossLink now = 300mm
       - All parts update geometry!
       - Assembly adjusts automatically!

    5. **Test motion again**

       - Move input 10mm
       - Output should now move 30mm (3√ó input)

    </Steps>

    ‚úÖ **This is parametric design power!**

  </TabItem>
  <TabItem label="Mathematical Verification">

    **Verify the math:**

    For a pantograph with scaling ratio k:

    ```
    If input point I is at (x, y) relative to O,
    Then output point P is at (k√óx, k√óy) relative to O
    ```

    **Example with k = 2:**
    - Input moves to I = (50, 50) mm
    - Output should be at P = (100, 100) mm

    **Test this:**

    <Steps>
    1. Position input at I = (50, 50)
    2. Measure P coordinates
    3. Verify P = (100, 100)
    </Steps>

    :::tip[Perfect Geometry]
    If your measurements match the math, your pantograph is geometrically perfect! The similar triangle relationships are working correctly.

## Common Issues and Solutions

### ``Output doesn't scale correctly"

- **Cause:** Incorrect link length ratios
- **Solution:** Verify expressions in spreadsheet
- **Fix:** InputLength must = BaseLength √ó k
- **Fix:** OutputLength must = BaseLength √ó k¬≤

### ``Mechanism binds or locks up"

- **Cause:** Links interfering, or over-constrained assembly
- **Solution:** Check clearances, ensure only Base is Fixed
- **Fix:** May need to offset arms in Z-direction

### ``Scaling ratio parameter doesn't update links"

- **Cause:** Dimensions not using formulas
- **Solution:** Check that InputLength cell has `=B3*B2` not a hardcoded value
- **Fix:** Recreate spreadsheet formulas

### ``Parts misalign in assembly"

- **Cause:** Not referencing master sketch consistently
- **Solution:** All part sketches must use External Geometry from same master sketch
- **Fix:** Rebuild parts referencing master sketch points
