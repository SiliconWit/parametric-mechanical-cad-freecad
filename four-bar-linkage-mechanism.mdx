<BionicText method="advanced" intensity="medium" class="highlight">
The four-bar linkage is one of the simplest yet most versatile planar mechanisms in mechanical engineering. It forms the backbone of countless mechanical systems, from vehicle suspensions to robotic arms, and understanding its design is fundamental to mechanism synthesis.
</BionicText>

### Real-World Applications

The four-bar linkage appears in countless engineering applications:

:::note[Common Applications]
- **Vehicle suspension systems** - Independent wheel movement with controlled geometry
- **Excavator arms and construction equipment** - Coordinated multi-axis motion
- **Door hinges and folding mechanisms** - Complex opening paths in tight spaces
- **Windshield wiper linkages** - Converting rotary motor motion to sweeping action
- **Robotic arm joints** - Precise positioning with parallel motion characteristics
- **Adjustable desk and chair mechanisms** - Ergonomic height adjustment systems
:::

### The Engineering Problem

> **Design Challenge:** Given four rigid links connected by revolute joints with one link fixed, how do we create controlled motion transformation with predictable path characteristics and full rotation capability?

## üìö Mechanism Fundamentals

### Components and Motion

A four-bar linkage consists of four elements working together:

<Tabs>
<TabItem label="Components">

**1. Ground Link (Link 1)**
- Fixed base providing reference frame
- Contains pivot points for moving links

**2. Input Link/Crank (Link 2)**
- Driven link, may rotate fully or oscillate
- Determines mechanism configuration

**3. Coupler (Link 3)**
- Connects crank and follower
- Points on coupler trace complex curves

**4. Output Link/Follower (Link 4)**
- Produces desired motion output
- May rotate or oscillate depending on design

</TabItem>
<TabItem label="Grashof's Theorem">

<Card title="üéì Grashof's Theorem" icon="star">

The behavior depends critically on link length relationships.

$$S + L \leq P + Q$$

Where:
- **S** = shortest link length
- **L** = longest link length
- **P, Q** = intermediate link lengths

</Card>

**If the inequality holds (Grashof linkage):**
- At least one link can rotate 360¬∞
- Configuration depends on which link is ground

**Which link is ground determines behavior:**
- **Crank-Rocker:** Crank rotates 360¬∞, follower oscillates
- **Double-Crank (Drag-link):** Both crank and follower rotate 360¬∞
- **Double-Rocker:** Both crank and follower oscillate

**If inequality doesn't hold (Non-Grashof):**
- No link can rotate fully; all oscillate

</TabItem>
<TabItem label="Design Example">

**For this lesson, we'll use:**

| Link | Length | Role |
|------|--------|------|
| Link 1 (ground) | 100 mm | Fixed base |
| Link 2 (crank) | 40 mm | Input rotator |
| Link 3 (coupler) | 80 mm | Connector |
| Link 4 (follower) | 70 mm | Output oscillator |

**Grashof Check:**

$$S = 40, L = 100, P = 70, Q = 80$$

$$S + L = 140$$

$$P + Q = 150$$

$$140 \leq 150 \checkmark$$

:::tip[Engineering Insight]
This is a **Grashof Crank-Rocker** configuration! Link 2 (crank) can rotate fully through 360¬∞, while Link 4 (follower) oscillates. This configuration is ideal for applications requiring continuous rotary input to produce oscillating output: exactly what windshield wipers need!
:::

</TabItem>
</Tabs>

## üí° Parametric Design Strategy

<BionicText method="advanced" intensity="medium" class="highlight">
We'll use a master sketch approach: defining the complete kinematic layout once, then referencing it from all parts. This is how professionals design mechanisms where geometric relationships between parts are more critical than individual part dimensions.
</BionicText>

### Our Design Approach

<Card title="üéØ Master Sketch Philosophy" icon="rocket">

We'll control the **entire four-bar mechanism** with one master sketch plus four length parameters:

**Link1** = 100 mm
**Link2** = 40 mm
**Link3** = 80 mm
**Link4** = 70 mm

Change the master sketch ‚Üí all parts update automatically!

This is the power of kinematic-first parametric design.

</Card>

### Design Workflow

<Steps>

1. Create Spreadsheet (parameter table)

2. Create Master Sketch (kinematic layout)

3. Create Link 1 - Ground (reference master)

4. Create Link 2 - Crank (reference master)

5. Create Link 3 - Coupler (reference master)

6. Create Link 4 - Follower (reference master)

7. Assemble with pin joints

8. Create technical drawing

9. Test Grashof configurations

</Steps>

:::note[Why Master Sketch First?]
By defining the **kinematic layout** before individual part geometry, we ensure all parts maintain correct positional relationships. The master sketch becomes the geometric truth: change joint positions once, every part adapts. This is essential for mechanism design where motion paths must remain coordinated.
:::

<TabItem label="Creating Aliases">

**Why Aliases?**

Aliases let you reference cells by name instead of "B2", making formulas readable and meaningful.

**Create aliases:**

<Steps>

1. Click cell **B2**
2. Right-click ‚Üí **Properties**
3. In "Alias" field, type: `Link1`
4. Click **OK**

5. Repeat for all parameter cells (B2 through B8):
   - B2: `Link1`
   - B3: `Link2`
   - B4: `Link3`
   - B5: `Link4`
   - B6: `LinkWidth`
   - B7: `LinkThickness`
   - B8: `PinRadius`

</Steps>

Now you can use `Spreadsheet.Link1` anywhere in your model!

</TabItem>
<TabItem label="Best Practices">

:::tip[Spreadsheet Best Practices]
- **Always use aliases** for values you'll reference throughout the model
- **Include units** in column C for documentation (they're comments, not calculations)
- **Use descriptive names** (Link1, not L1 or a)
- **Keep it organized** - group related parameters together
- **Add notes** in additional columns to document design decisions and Grashof calculations
:::

## üîß Part 4: Creating the Master Sketch

<InArticleAd />


### What is a Master Sketch?

<Card title="üéØ Master Sketch Concept" icon="star">

A **master sketch** defines the kinematic layout: where joints are located and how links connect. All parts reference this sketch geometry as their positional truth.

**Key Benefits:**
- Change joint positions once ‚Üí all parts update
- Ensures geometric consistency across all links
- Makes mechanism reconfiguration effortless

</Card>

### Creating the Master Sketch

<Steps>

1. **Ensure you're in Part Design workbench**

2. **Create new sketch on XY_Plane**

   - Click **Create Sketch** button
   - Select **XY_Plane**
   - Click **OK**

   This sketch won't belong to a body; it's standalone reference geometry!

</Steps>

### Drawing the Linkage Layout

<Tabs>
<TabItem label="Ground Link (Link 1)">

**Draw a horizontal line:**

<Steps>

1. **Line tool** (press L)

2. Click at **origin** (0, 0)

3. Move horizontally to the right

4. Click to place endpoint

5. Press **Escape**

6. **Make it horizontal:**
   - Select the line
   - Press **H** key

7. **Fix the start point:**
   - Coincident constraint
   - Click left endpoint
   - Click origin

8. **Dimension the link:**
   - Distance constraint
   - Click both endpoints
   - Click **∆íx** button
   - Formula: `Spreadsheet.Link1`
   - Press **Enter**

</Steps>

:::note[Joint Labels]
Label this mentally as "ground link" with:
- **Joint A** (left endpoint at origin)
- **Joint B** (right endpoint)
:::

</TabItem>
<TabItem label="Crank (Link 2)">

**Draw a line from point A:**

<Steps>

1. **Line tool** (L)

2. Start at the **left endpoint** of Link 1 (point A)

3. Draw at an angle upward-right

4. Click to place endpoint

5. Press **Escape**

6. **Constrain the link:**
   - Coincident constraint
   - Click start of new line
   - Click left endpoint of Link 1 (point A)

7. **Dimension the length:**
   - Distance constraint
   - Click both endpoints of the new line
   - Click **∆íx** button
   - Formula: `Spreadsheet.Link2`
   - Press **Enter**

</Steps>

This is the **crank**, connecting joints **A** and **C**.

</TabItem>
<TabItem label="Follower (Link 4)">

**Draw a line from point B:**

<Steps>

1. **Line tool** (L)

2. Start at **right endpoint** of Link 1 (point B)

3. Draw upward at an angle

4. Click to place endpoint

5. Press **Escape**

6. **Constrain:**
   - Coincident at point B
   - Distance = `Spreadsheet.Link4`

</Steps>

This is the **follower**, connecting joints **B** and **D**.

</TabItem>
<TabItem label="Coupler (Link 3)">

**Draw a line connecting the free ends:**

<Steps>

1. **Line tool** (L)

2. Start at free end of Link 2 (point **C**)

3. End at free end of Link 4 (point **D**)

4. Press **Escape**

5. **Constrain:**
   - Coincident at point C
   - Coincident at point D
   - Distance = `Spreadsheet.Link3`

</Steps>

:::tip[Success!]
You now have a closed four-bar linkage layout! All four links form a connected chain.
:::

</TabItem>
<TabItem label="Construction Geometry">

**Make sketch reference-only:**

<Steps>

1. **Select all four lines**
   - Press **Ctrl+A** or box-select all lines

2. **Toggle construction geometry**
   - Press **G** key
   - Lines turn blue/dashed

</Steps>

<Card title="üéØ Why Construction Geometry?" icon="star">

**Construction geometry is reference only:**
- Won't create solid features
- Parts can reference these lines without inheriting them
- Perfect for kinematic layouts and design intent

This is the key to master sketch workflows!

</Card>

</TabItem>
<TabItem label="Finalize">

**Check and close:**

<Steps>

1. **Check constraints:**

   The sketch should be fully constrained. Verify:
   - Link 1: horizontal, one end at origin, length ‚úì
   - Link 2: coincident at A, length ‚úì
   - Link 3: coincident at C and D, length ‚úì
   - Link 4: coincident at B, length ‚úì

2. **Optional: Add angle dimension**

   If under-constrained, add angle to Link 2:
   - Angle constraint
   - Link 2 to horizontal
   - Type: `30` degrees

3. **Close the sketch**

   Click **Close** button in toolbar

</Steps>

:::note[Master Sketch Complete!]
You now have the kinematic foundation for your entire four-bar linkage mechanism. This single sketch will control all four parts!
:::

</TabItem>
</Tabs>

## üî© Part 5: Creating Link 1 (Ground Link)

### Design Intent

<Card title="‚öôÔ∏è Link 1 Requirements" icon="document">

Link 1 is a rectangular plate with:

- **Two circular holes** at joints A and B for pivot pins
- **Parametric dimensions** controlled by spreadsheet
- **References master sketch** for joint positions

</Card>

### Creating Link 1

<Steps>

1. **Create a Body**

   - Click **Create Body** button
   - Right-click Body ‚Üí **Rename**
   - Type: `Link1_Ground`

2. **Create a Sketch**

   - Select Link1_Ground body
   - Create sketch on **XY_Plane**

</Steps>

### Drawing the Link Plate

<Tabs>
<TabItem label="External References">

**Reference master sketch geometry:**

<Steps>

1. **External geometry tool** (press E)

2. Click the **master sketch** in the tree

3. Select **joint A endpoint** (origin)

4. Select **joint B endpoint** (right end of Link 1)

5. Press **Escape**

</Steps>

:::tip[Purple Reference Geometry]
The selected points now appear as **purple geometry** in your sketch. These are external references that track the master sketch!
:::

</TabItem>
<TabItem label="Pin Holes">

**Draw circles at both joints:**

<Steps>

1. **Circle tool** (C)

2. Draw circle centered at **origin** (joint A)

3. Press **Escape**

4. **Circle tool** again

5. Draw circle at **joint B** reference point

6. Press **Escape**

7. **Constrain both circles:**
   - Coincident: First circle center ‚Üí origin reference
   - Coincident: Second circle center ‚Üí joint B reference

8. **Dimension the holes:**
   - Radius constraint on first circle
   - Click **∆íx** ‚Üí `Spreadsheet.PinRadius`
   - Repeat for second circle

</Steps>

</TabItem>
<TabItem label="Link Body">

**Create the connecting plate:**

<Steps>

1. **Rectangle tool**

2. Draw rectangle connecting the two circles

3. Press **Escape**

4. **Symmetric constraint:**
   - Select top edge of rectangle
   - Select bottom edge of rectangle
   - Select horizontal centerline between holes
   - This centers the rectangle on the link axis!

5. **Dimension width:**
   - Distance constraint (perpendicular to link)
   - Click **∆íx** ‚Üí `Spreadsheet.LinkWidth`

</Steps>

</TabItem>
<TabItem label="Finalize">

**Complete the sketch:**

<Steps>

1. **Verify fully constrained**

   Check solver messages panel

2. **Close sketch**

   Click **Close** button

3. **Pad the link**

   - Select the sketch
   - Click **Pad** tool
   - Length: Click **∆íx** ‚Üí `Spreadsheet.LinkThickness`
   - Click **OK**

</Steps>

:::note[Link 1 Complete!]
You now have a parametric ground link that references the master sketch for joint positions!
:::

</TabItem>
</Tabs>

## üß© Part 9: Assembly

<BionicText method="advanced" intensity="medium" class="highlight">
Assembly is where your individual linkage plates come together as a functioning mechanism. By applying pin joint constraints, you'll see Grashof's Theorem in action as the crank rotates and the follower oscillates!
</BionicText>

### Assembly Strategy

<Card title="üéØ Constraint Plan" icon="rocket">

1. **Link 1 (Ground):** Fixed; provides reference frame
2. **Link 2 (Crank):** Rotates about joint A
3. **Link 3 (Coupler):** Pivots at joints C and D
4. **Link 4 (Follower):** Pivots at joints B and D (oscillates)

All joints use **Axial Align** constraints to allow rotation!

</Card>

### Creating the Assembly

<Tabs>
<TabItem label="Setup">

<Steps>

1. **Switch to Assembly workbench**

   Use workbench dropdown

2. **Create new assembly**

   Assembly ‚Üí Create Assembly

3. **Add all four parts**

   Drag from tree or use "Add Part" button:
   - Link1_Ground
   - Link2_Crank
   - Link3_Coupler
   - Link4_Follower

</Steps>

</TabItem>
<TabItem label="Ground Link">

<Steps>

1. Select **Link1_Ground** in assembly tree

2. Click **Fixed constraint** button

3. Link 1 is now locked in place (ground reference)

</Steps>

:::note[Why Fix Link 1?]
Every mechanism needs a ground reference. Link 1 provides the fixed frame from which all other motion is measured. This is fundamental to four-bar linkage analysis!
:::

</TabItem>
<TabItem label="Pin Joints">

**Apply Axial Align constraints at all four joints:**

<Steps>

1. **Joint A (Link 1 - Link 2):**
   - Select hole axis from Link 1 at point A
   - Select hole axis from Link 2 at point A
   - Click **Axial Align** constraint

2. **Joint B (Link 1 - Link 4):**
   - Link 1 hole at B
   - Link 4 hole at B
   - Axial Align

3. **Joint C (Link 2 - Link 3):**
   - Link 2 hole at C
   - Link 3 hole at C
   - Axial Align

4. **Joint D (Link 3 - Link 4):**
   - Link 3 hole at D
   - Link 4 hole at D
   - Axial Align

</Steps>

</TabItem>
<TabItem label="Testing">

**Test the mechanism:**

<Steps>

1. **Drag Link 2** (the crank) in the assembly view

2. **Observe motion:**
   - Link 2 should rotate smoothly through 360¬∞
   - Link 3 follows with complex motion
   - Link 4 oscillates back and forth!

</Steps>

:::tip[Grashof's Theorem in Action!]
You're witnessing a **Crank-Rocker mechanism**! Link 2 (shortest adjacent to ground) rotates fully, while Link 4 (follower) oscillates. This is exactly what Grashof's Theorem predicted!
:::

**Troubleshooting:**

:::caution[If Mechanism Won't Move]
- Check that **only Link 1** has Fixed constraint
- All pin joints should be **Axial Align** (not Fixed!)
- Remove any redundant constraints
- Ensure proper hole alignment
:::

</TabItem>
</Tabs>

## ‚ùì Common Issues and Solutions

<Tabs>
<TabItem label="Assembly Issues">

**"Parts don't align in assembly"**

:::caution[Solution]
- Ensure you referenced the **same master sketch points** in all part sketches
- Check that pin hole radii are **identical** where links connect
- Verify External Geometry references are correct
:::

**"Mechanism is locked/won't move"**

:::caution[Solution]
- Check that **only Link 1** has Fixed constraint
- All other links should have **only Axial Align** constraints at pins
- Too many constraints will over-constrain the system
- Remove any accidental Fixed constraints on moving parts
:::

</TabItem>
<TabItem label="Sketch Issues">

**"Master sketch is under-constrained"**

:::caution[Solution]
- Each link needs **length dimension**
- Need one **angle dimension** to set initial configuration
- One point must be **fixed** (origin)
- Check solver messages for missing constraints
:::

**"External geometry not updating"**

:::caution[Solution]
- Ensure you used **External Geometry tool** (E), not just coincident
- Master sketch must be **closed** before referencing
- Try recomputing master sketch first, then parts
:::

</TabItem>
<TabItem label="Parametric Issues">

**"Parts don't update with spreadsheet"**

:::caution[Solution]
- Check you used **formulas** (`Spreadsheet.Link1`), not hardcoded values
- Verify **aliases** were created in spreadsheet Properties
- Check dimension shows **∆íx** icon
- Manually recompute individual bodies
:::

</TabItem>
</Tabs>
