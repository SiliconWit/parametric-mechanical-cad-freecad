<BionicText method="advanced" intensity="medium" class="highlight">
Scissor lift mechanisms provide vertical motion through a compact, stable structure using multiple interconnected linkages. They're fundamental to many height-adjustment and material handling applications, converting small horizontal input motion into amplified vertical displacement.
</BionicText>

### Real-World Applications

The scissor lift appears everywhere in engineering:

:::note[Common Applications]
- **Vehicle scissor jacks** - Compact emergency lifting for tire changes
- **Aerial work platforms** - Construction and maintenance access at height
- **Adjustable work tables** - Ergonomic height adjustment in manufacturing
- **Stage platforms** - Theatrical equipment for dynamic staging
- **Medical examination tables** - Patient positioning with smooth vertical control
- **Warehouse loading platforms** - Material handling and dock leveling
:::

### The Engineering Problem

> **Design Challenge:** Given limited horizontal space, how do we achieve significant vertical lifting with a stable, compact mechanism that stores efficiently when collapsed?

## üìö Mechanism Fundamentals

### Components and Motion

A scissor mechanism consists of crossed links working in harmony:

<Tabs>
  <TabItem label="Components">

    **1. Scissor Arms (Links)**
    - Crossed in X-shaped pairs
    - Multiple pivot points per arm
    - Connected in series for stages

    **2. Pivot Pins**
    - End pivots connect to base/top
    - Center pivot where arms cross
    - Allow relative rotation

    **3. Base Platform**
    - Fixed reference frame
    - Mounts bottom pivots
    - Provides stability

    **4. Top Platform**
    - Moving platform
    - Carries the load
    - Rises with mechanism extension

</TabItem>
  <TabItem label="Key Parameters">

    <Card title="‚öôÔ∏è Critical Design Parameters" icon="setting">

    **Arm length (L)**
    - Length of each scissor arm
    - Directly affects maximum height
    - Typical range: 150-300mm for small lifts

    **Number of stages (N)**
    - How many X-pairs in series
    - Multiplies lift height
    - More stages = more compact when collapsed

    **Minimum angle (Œ∏_min)**
    - Collapsed position angle
    - Affects minimum height
    - Typically 15-20 degrees

    **Maximum angle (Œ∏_max)**
    - Extended position angle
    - Affects maximum height
    - Typically 60-70 degrees

    </Card>

</TabItem>
  <TabItem label="Motion Analysis">

    **Height Relationship:**
    ```
    Collapsed height: H_min = 2N √ó L √ó sin(Œ∏_min)
    Extended height:  H_max = 2N √ó L √ó sin(Œ∏_max)
    Lift ratio:       H_max / H_min = sin(Œ∏_max) / sin(Œ∏_min)
    ```

    **Design Guidelines:**
    - N ‚â• 2 for significant lift height
    - L/W ratio ‚â• 4 for stable geometry
    - Œ∏_max ‚â§ 70¬∞ to avoid instability

    :::tip[Engineering Insight]
    The scissor lift's mechanical advantage varies with angle. At shallow angles (collapsed), it provides maximum force amplification but requires more horizontal input. At steep angles (extended), it moves faster vertically but with less force multiplication. This is why scissor jacks feel easier at the start!
    :::

</TabItem>
</Tabs>

## üöÄ Part 1: Design Strategy and Planning

### Understanding Scissor Geometry

<BionicText method="advanced" intensity="medium" class="highlight">
The scissor lift uses the same arm design multiple times. This repeating geometry pattern is key to scalable parametric design: design once, use everywhere, control centrally.
</BionicText>

### Our Design Approach

<Card title="üéØ Repeating Geometry Philosophy" icon="rocket">

We'll control the **entire scissor lift** with parametric design:

**ArmLength** = 200 mm
**ArmWidth** = 30 mm
**StageCount** = 2 (for documentation)

Design ONE perfect arm ‚Üí Use it four times (for 2-stage lift)!

This is industrial-strength parametric methodology.

</Card>

### Design Workflow

```
Step 1: Create Spreadsheet (parameter table)
Step 2: Design Scissor Arm (one link, three holes)
Step 3: Create Base Platform (mount points)
Step 4: Create Top Platform (load carrier)
Step 5: Assemble single stage with constraints
Step 6: Test parametric control
Step 7: Create technical drawing
Step 8: Extension - add second stage
```

:::note[Why Symmetric Design Matters]
Scissor mechanisms rely on perfect symmetry. By constraining our sketch symmetrically about a centerline and using midpoint constraints, we ensure the mechanism will function correctly no matter how we change the parameters.
:::

## üî© Part 3: Creating the Scissor Arm

<InArticleAd />


### Design Intent

<Card title="‚öôÔ∏è Scissor Arm Requirements" icon="document">

The scissor arm is a rectangular link with three holes:

- **End hole 1** (left) - Connects to base or adjacent stage
- **Center hole** (middle) - Where arms cross (intersection pivot)
- **End hole 2** (right) - Connects to top platform or adjacent stage
- **Parametric control** - All dimensions driven by spreadsheet

</Card>

### Step-by-Step: Scissor Arm Part

<Steps>

1. **Create a Body**

   - Ensure you're in **Part Design workbench**
   - Click **Create Body** button (or Body ‚Üí Create body)
   - A "Body" object appears in the tree
   - Right-click ‚Üí Rename ‚Üí type `ScissorArm`

2. **Create a Sketch**

   - Select the "ScissorArm" body in tree
   - Click **Create Sketch** button
   - Dialog asks: Choose a plane
   - Select **XY_Plane**
   - Click **OK**

   You're now in **Sketcher workbench** (automatic switch)

</Steps>

### Drawing the Centerline Reference

<Tabs>
  <TabItem label="Construction Line">

    **Draw horizontal reference:**

    <Steps>

    1. **Line tool** (press L)

    2. Click at **origin** (0, 0)

    3. Move horizontally to the **right**

    4. Click to place endpoint

    5. Press **Escape**

    6. Select the line

    7. **Make it horizontal:** Press **H** key

    </Steps>

    You now have a horizontal line from the origin!

</TabItem>
  <TabItem label="Parametric Length">

    **Dimension the centerline:**

    <Steps>

    1. **Distance constraint** tool

    2. Click the **left endpoint** of the line

    3. Click the **right endpoint** of the line

    4. A dimension dialog appears

    5. Click the small **∆íx** button

    6. In the formula editor, type: `Spreadsheet.ArmLength`

    7. Press **Enter**

    </Steps>

    Line is now 200mm and parametric!

</TabItem>
  <TabItem label="Make Construction">

    **Convert to construction geometry:**

    <Steps>

    1. **Select the line** (click on it)

    2. Press **G** key (or click Construction Mode button)

    3. The line turns **blue/dashed**

    </Steps>

    :::note[What is Construction Geometry?]
    Construction geometry is reference-only: it helps constrain your design but won't become part of the final 3D shape. Perfect for centerlines, reference axes, and symmetry lines!
    :::

</TabItem>
</Tabs>

### Adding the Three Pin Holes

<Tabs>
  <TabItem label="Center Hole">

    **Draw the crossing pivot:**

    <Steps>

    1. **Circle tool** (press C)

    2. Click somewhere near the **middle** of the line

    3. Move mouse out and click to set size

    4. Press **Escape**

    5. **Midpoint constraint:**
       - Click **Midpoint constraint** tool
       - Click the **circle center**
       - Click the **centerline** (the construction line)
       - Circle center snaps to line midpoint!

    6. **Radius dimension:**
       - Radius constraint tool
       - Click the circle
       - Click **∆íx** button
       - Type: `Spreadsheet.PinRadius`
       - Enter

    </Steps>

    Center hole is perfectly positioned and parametric!

</TabItem>
  <TabItem label="Left End Hole">

    **Draw the base connection:**

    <Steps>

    1. **Circle tool**

    2. Click near the **left end** of the centerline

    3. Click to set size

    4. Press **Escape**

    5. **Coincident constraint:**
       - Coincident tool
       - Click **circle center**
       - Click **left endpoint** of centerline
       - Circle locks to origin!

    6. **Radius:**
       - Radius constraint
       - Click the circle
       - ∆íx ‚Üí `Spreadsheet.PinRadius`
       - Enter

    </Steps>

    Left hole positioned at origin!

</TabItem>
  <TabItem label="Right End Hole">

    **Draw the top platform connection:**

    <Steps>

    1. **Circle tool**

    2. Click near the **right end** of centerline

    3. Click to set size

    4. Press **Escape**

    5. **Coincident constraint:**
       - Circle center ‚Üí Right endpoint of centerline

    6. **Radius:**
       - ∆íx ‚Üí `Spreadsheet.PinRadius`

    </Steps>

    All three holes are now positioned and dimensioned!

</TabItem>
</Tabs>

### Creating the Arm Body

<Tabs>
  <TabItem label="Rectangle Profile">

    **Draw the structural beam:**

    <Steps>

    1. **Rectangle tool**

    2. Draw a rectangle that encompasses all three circles

    3. Should extend past the leftmost and rightmost circles

    4. Press **Escape**

    </Steps>

    Don't worry about exact positioning yet. Constraints will fix it!

</TabItem>
  <TabItem label="Symmetric Constraint">

    **Center the rectangle about centerline:**

    <Steps>

    1. Click **Symmetric constraint** tool

    2. Click the **top edge** of the rectangle

    3. Click the **bottom edge** of the rectangle

    4. Click the **horizontal centerline** (construction line)

    5. Press **Escape**

    </Steps>

    The rectangle is now perfectly centered vertically!

    :::tip[Symmetry is Powerful]
    The symmetric constraint maintains equal distance above and below the centerline. When you change ArmWidth, the rectangle expands equally in both directions. This is parametric design at its finest!
    :::

</TabItem>
  <TabItem label="Width Dimension">

    **Set the parametric width:**

    <Steps>

    1. **Distance constraint** tool

    2. Click the **top edge** of rectangle

    3. Click the **bottom edge** of rectangle

    4. ∆íx button

    5. Type: `Spreadsheet.ArmWidth`

    6. Enter

    </Steps>

    Arm width is now 30mm and parametric!

</TabItem>
  <TabItem label="Length Lock">

    **The rectangle length is automatically constrained!**

    Because the rectangle corners will be coincident with or near the end hole centers (which are already positioned at the centerline endpoints), the rectangle length is implicitly defined.

    If the solver shows degrees of freedom, add:

    - **Coincident:** Rectangle left edge ‚Üí passes through left hole center
    - **Coincident:** Rectangle right edge ‚Üí passes through right hole center

    Or simply ensure the rectangle extends to the endpoints.

</TabItem>
</Tabs>

### Finalizing the Sketch

<Steps>

1. **Check the solver**

   Look at "Solver Messages" panel:
   - Should say **"Fully constrained"**
   - If it says "X degrees of freedom", review constraints

2. **Verify the design**

   You should see:
   - Horizontal construction line (blue/dashed)
   - Three circles on the line
   - Rectangle centered about the line
   - All dimensions showing formulas

3. **Close the sketch**

   Click the **Close** button in the toolbar

</Steps>

<Tabs>
  <TabItem label="Applying Pad">

    <Steps>

    1. **Select the sketch** in the tree (under ScissorArm body)

    2. Click **Pad** tool in Part Design toolbar

    3. In the Pad panel (left side):
       - Type: **Dimension**
       - Click the **∆íx** button next to Length
       - Type: `Spreadsheet.ArmThickness`
       - Click **OK**

    4. Click **OK** to complete the pad

    </Steps>

    **You now have a 3D scissor arm!**

</TabItem>
  <TabItem label="Viewing Your Part">

    **Navigate the 3D view:**

    - Press **V** then **F** to fit all
    - Rotate view with middle mouse button
    - See the rectangular link with three holes!

    :::note[What You Created]
    - A parametric 3D scissor link
    - Three precision-positioned pin holes
    - Perfectly symmetric about centerline
    - Controlled by spreadsheet
    - Ready for multiple use in assembly
    :::

</TabItem>
</Tabs>

## üì¶ Part 4: Creating the Base Platform

### Design Intent

The base provides stable mounting for the bottom pivots of the scissor mechanism.

<Card title="üèóÔ∏è Base Platform Requirements" icon="puzzle">

- **Wide footprint** - Stability when mechanism extends
- **Pin mount holes** - For bottom pivots of arms
- **Parametric width** - Controlled by `Spreadsheet.BaseWidth`
- **Appropriate thickness** - Structural integrity

</Card>

### Creating the Base

<Steps>

1. **Create new Body**

   - Part Design workbench
   - Create Body
   - Rename: `BasePlatform`

2. **Create Sketch**

   - Select BasePlatform body
   - Create Sketch ‚Üí XY_Plane

</Steps>

### Drawing the Base Platform

<Tabs>
  <TabItem label="Main Rectangle">

    <Steps>

    1. **Rectangle tool**

    2. Draw rectangle roughly centered at origin

    3. Press **Escape**

    4. **Symmetric constraints:**
       - Symmetric: left edge, right edge, Y-axis
       - Symmetric: top edge, bottom edge, X-axis
       - This centers the rectangle perfectly!

    5. **Width dimension:**
       - Distance: left edge to right edge
       - ∆íx ‚Üí `Spreadsheet.BaseWidth`
       - Enter

    6. **Depth dimension:**
       - Distance: top edge to bottom edge
       - Type: `60` mm (fixed depth)
       - Enter

    </Steps>

</TabItem>
  <TabItem label="Pin Mount Holes">

    For a single-stage lift, we need two pin holes for the bottom pivots.

    **Left pin hole:**

    <Steps>

    1. **Circle tool**

    2. Draw circle on the left side

    3. Press **Escape**

    4. **Horizontal constraint:**
       - Circle center and origin (centers on X-axis)

    5. **Distance from origin:**
       - Distance: origin to circle center
       - Type: `80` mm (horizontal spacing)
       - Enter

    6. **Radius:**
       - ∆íx ‚Üí `Spreadsheet.PinRadius`

    </Steps>

    **Right pin hole:**

    <Steps>

    1. Draw another circle on the right

    2. **Horizontal constraint:** Circle center and origin

    3. **Distance from origin:** `80` mm

    4. **Radius:** ∆íx ‚Üí `Spreadsheet.PinRadius`

    </Steps>

    Or better - use **Equal and Symmetric** constraints:

    <Steps>

    1. After creating both circles
    2. **Equal constraint:** Both circles (equal radii)
    3. **Symmetric constraint:** Both circle centers, Y-axis
    4. **Distance:** One circle center to Y-axis = `80` mm

    </Steps>

    This ensures perfect symmetry!

</TabItem>
  <TabItem label="Check & Finish">

    **Verify and complete:**

    :::caution[Check Solver]
    Solver should show: **"Fully constrained"**

    The sketch should have:
    - Rectangle centered at origin
    - Two symmetric pin holes
    - All parametric dimensions in place
    :::

    <Steps>
    1. Confirm "Fully constrained"
    2. **Close sketch**
    </Steps>

</TabItem>
</Tabs>

### Padding the Base

<Steps>

1. **Select the sketch** in the tree

2. **Pad tool**

3. Length: ∆íx ‚Üí `Spreadsheet.BaseThickness`

4. **OK**

</Steps>

**Base platform complete!**

## üß© Part 6: Assembly - Single Stage

<InArticleAd />


<BionicText method="advanced" intensity="medium" class="highlight">
Assembly is where your individual parts come together as a functioning mechanism. The scissor lift assembly uses the same arm part twice in a crossed configuration, demonstrating how parametric parts can be reused in different orientations.
</BionicText>

### Assembly Strategy

<Card title="üéØ Single-Stage Assembly Plan" icon="rocket">

1. **Base Platform:** Fixed (ground reference)
2. **First Scissor Arm:** Left-to-right rising diagonal
3. **Second Scissor Arm:** Right-to-left rising diagonal
4. **Center crossing:** Arms pivot where they cross
5. **Top Platform:** Rises with mechanism extension

</Card>

### Creating the Assembly

<Tabs>
  <TabItem label="Setup">

    <Steps>

    1. **Switch to Assembly workbench**

       Use workbench dropdown (Assembly4 or A2plus recommended)

    2. **Create new assembly**

       Assembly ‚Üí Create Assembly

    3. **Add parts**

       You need to add:
       - BasePlatform (once)
       - ScissorArm (twice - same part, two instances!)
       - TopPlatform (once)

       Use "Add Part" or drag from tree, adding ScissorArm twice

    </Steps>

</TabItem>
  <TabItem label="Fixing Base">

    <Steps>

    1. Select **BasePlatform** in assembly tree

    2. Click **Fixed constraint** button

    3. Base is now locked in place (ground link)

    </Steps>

    :::note[Why Fix the Base?]
    Every mechanism needs a ground reference. The base platform provides this fixed foundation from which all vertical motion is measured.
    :::

</TabItem>
  <TabItem label="First Arm Position">

    **Position the left-rising arm:**

    <Steps>

    1. Select the **left end hole axis** of the first ScissorArm

    2. Select the **left pin hole axis** of the BasePlatform

    3. Click **Axial Align constraint**

    </Steps>

    The first arm's left end is now connected to the base!

    :::tip[Naming Parts]
    Rename your ScissorArm instances in the assembly to "ScissorArm_Left" and "ScissorArm_Right" to avoid confusion!
    :::

</TabItem>
  <TabItem label="Second Arm Position">

    **Position the right-rising arm:**

    <Steps>

    1. Select the **left end hole axis** of the second ScissorArm

    2. Select the **right pin hole axis** of the BasePlatform

    3. Click **Axial Align constraint**

    </Steps>

    The second arm's left end connects to the right side of the base!

</TabItem>
  <TabItem label="Center Crossing">

    **This is the critical scissor pivot!**

    <Steps>

    1. Select the **center hole axis** of first ScissorArm

    2. Select the **center hole axis** of second ScissorArm

    3. Click **Axial Align constraint**

    </Steps>

    The arms now cross and can pivot relative to each other!

    :::caution[Assembly Critical Point]
    This center crossing constraint is what makes it a "scissor" mechanism. Without this, the arms would be independent. With it, they work together to amplify motion!
    :::

</TabItem>
  <TabItem label="Top Platform">

    **Connect to the top platform:**

    <Steps>

    1. First arm's **right end hole** ‚Üí TopPlatform **right pin hole**
       - Axial Align constraint

    2. Second arm's **right end hole** ‚Üí TopPlatform **left pin hole**
       - Axial Align constraint

    </Steps>

    The mechanism is now complete!

</TabItem>
  <TabItem label="Testing Motion">

    **Test the scissor action:**

    Try dragging the top platform or an arm in the assembly view:
    - Arms should rotate about their pivots
    - Center crossing should move
    - Top platform should rise and fall!
    - Motion should be smooth and coordinated

    :::tip[Troubleshooting Motion]
    **If mechanism won't move:**
    - Check that only BasePlatform is Fixed
    - All pin joints should be Axial Align (not Fixed!)
    - Remove any redundant constraints
    - Verify you have 4 arm-to-platform connections + 1 crossing connection
    :::

</TabItem>
</Tabs>

### Creating Professional Documentation

<Steps>

1. **Switch to TechDraw workbench**

2. **Create a page:**

## ‚ùì Common Issues and Solutions

<Tabs>
  <TabItem label="Sketch Issues">

    **"Center hole not at midpoint"**

    :::caution[Solution]
    - Use **Midpoint constraint**, not approximate positioning
    - Select circle center ‚Üí Select construction line
    - Apply Midpoint constraint from toolbar
    - Verify constraint icon appears on line
    :::

    **"Rectangle not symmetric"**

    :::caution[Solution]
    - Use **Symmetric constraint** on edges
    - Select top edge, bottom edge, then centerline
    - Order matters: two objects, then reference line
    - Check that centerline is construction geometry
    :::

</TabItem>
  <TabItem label="Parametric Issues">

    **"PinRadius doesn't update with PinDiameter"**

    :::caution[Solution]
    - Check cell B6 contains formula: `=B5/2`
    - Verify alias is on B6, not B5
    - Delete and recreate the formula if needed
    - Recompute after spreadsheet changes
    :::

    **"Arms don't scale uniformly"**

    :::caution[Solution]
    - Verify all dimensions use formulas (∆íx icon visible)
    - Check for hardcoded values that should be parametric
    - Ensure both arm instances reference same source part
    - Try "Mark to Recompute" on body, then Ctrl+R
    :::

</TabItem>
  <TabItem label="Assembly Issues">

    **"Arms don't cross properly"**

    :::caution[Solution]
    - Verify center hole exists on both arms
    - Check Axial Align constraint on center holes
    - Ensure arms are oriented correctly (one rises left, one rises right)
    - May need to flip one arm orientation
    :::

    **"Platform holes don't match arm holes"**

    :::caution[Solution]
    - All holes must use same `Spreadsheet.PinRadius`
    - Check spacing matches collapsed geometry
    - For single stage: platform holes ~80mm from center
    - Verify all radius constraints reference parameter
    :::

    **"Can't add ScissorArm twice"**

    :::caution[Solution]
    - Use assembly workbench "Add Part" button twice
    - Or right-click body ‚Üí "Duplicate Object" before assembly
    - Assembly4: Use "Insert Link" for same part multiple times
    - A2plus: Import the same part file twice
    :::

</TabItem>
</Tabs>
